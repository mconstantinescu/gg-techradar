About the Technology Radar

The Technology Radar is a landscape view of technologies the organisation tracks, evaluates, and recommends. Inspired by the ThoughtWorks Technology Radar, it provides a visual snapshot of the technology ecosystem at Gunvor Group — helping engineering teams make informed choices about what to adopt, trial, or avoid.

How It Works

The radar organises technologies across two axes:

  • Quadrants — four technology domains that group related technologies together.
  • Rings — four adoption levels that indicate how strongly we recommend each technology.

Quadrants

Methods & Patterns

Architectural approaches, design patterns, and engineering practices.

Tools

Build tools, CI/CD, testing frameworks, and developer utilities.

Platforms & Services

Cloud platforms, infrastructure services, and managed offerings.

Languages & Frameworks

Programming languages, application frameworks, and runtime environments.

Rings (Adoption Levels)

Ring Meaning
Adopt We strongly recommend this technology for broad use. It's proven, well-supported, and aligned with our strategy.
Trial Worth pursuing. Use on projects that can handle the risk of early adoption. Teams should share findings.
Assess Under evaluation. Worth exploring to understand its potential impact. Not yet for production use.
Hold Proceed with caution. Not recommended for new projects. Existing use should be reviewed for migration.

Flags

Each blip on the radar may carry a flag:

  • New — first appearance on the radar in this edition.
  • Changed — moved to a different ring since the last edition.

Connected Sections

The radar is one part of the Technology Portal's governance framework:

  • Decision Records — why we chose specific technologies (TDR/ADR).
  • Documentation — how to implement and operate them (blueprints, golden paths, SOPs).

Together: Radar (what) → Decisions (why) → Docs (how).

Contributing

To propose a new technology blip or update an existing one:

  1. Create or edit a markdown file in the latest edition folder of the Radar repository.
  2. Set the appropriate ring and quadrant in the frontmatter.
  3. Open a pull request — the Technology Advisory Group reviews radar changes.
